# RunPod 資料下載與實驗總流程（完整版本）

本文件把本專案提到的公開資料來源分成兩類：

1. **本專案已直接支援的主線資料**：DROID、FINO
2. **論文擴充與對照可參考的資料**：BridgeData V2、LIBERO、CALVIN、robomimic、REASSEMBLE、AHA/FailGen

> 重要：目前程式碼**直接整合**的是 **DROID → shards → train-sharded** 與 **FINO manifest → shards → fine-tune-fino**。其他資料來源已補上**官方下載與放置教學**，但尚未內建一鍵轉檔器。

## 1. RunPod 上的標準目錄

所有下載、轉檔與輸出都建議放在 `/workspace`：

```text
/workspace/
├── edsvfh_public_v2/
├── data/
│   ├── raw/
│   │   ├── droid/
│   │   ├── fino/
│   │   ├── bridgedata_v2/
│   │   ├── libero/
│   │   ├── calvin/
│   │   └── robomimic/
│   └── converted/
├── artifacts/
└── logs/
```

## 2. DROID（本專案主線）

### 2.1 官方資料來源
- full RLDS：`gs://gresearch/robotics/droid`
- debug 子集：`gs://gresearch/robotics/droid_100`
- raw：`gs://gresearch/robotics/droid_raw`

### 2.2 RunPod 上下載 debug 子集

```bash
bash scripts/runpod/01_download_droid100.sh
```

下載完成後資料會在：

```text
/workspace/data/raw/droid/droid_100
```

### 2.3 轉成本專案第二章使用的 shards

```bash
bash scripts/runpod/02_convert_droid100.sh
```

輸出位置：

```text
/workspace/data/converted/droid_debug
```

### 2.4 用 full DROID（正式主實驗）

先小量 smoke test：

```bash
export DROID_SOURCE=gs://gresearch/robotics/droid/1.0.1
export MAX_EPISODES=32
bash scripts/runpod/04_convert_droid_curated.sh
bash scripts/runpod/05_train_droid_curated.sh
```

如果 smoke test 通，再放大到 128 / 256 / 512。

## 3. FINO / FAILURE（本專案主線）

FINO 官方 repo 提供兩個下載入口：
- Download Data
- Download Annotations

由於官方資料頁的實際壓縮檔命名與解壓結構可能變動，本專案採用**manifest 轉換路線**，以降低資料樹變化對程式的影響。

### 3.1 建議放置位置

```text
/workspace/data/raw/fino
```

### 3.2 產生 manifest

如果你的 FINO 是 episode tree 形式，可直接掃描：

```bash
bash scripts/runpod/06_generate_fino_manifest.sh
```

輸出：

```text
/workspace/data/raw/fino/fino_manifest.jsonl
```

### 3.3 轉成 shards

```bash
bash scripts/runpod/07_convert_fino.sh
```

### 3.4 做 failure-horizon fine-tune

```bash
bash scripts/runpod/08_finetune_fino.sh
```

## 4. BridgeData V2（擴充來源，未內建轉檔）

官方下載入口：
- raw demos：`https://rail.eecs.berkeley.edu/datasets/bridge_release/data/demos_8_17.zip`
- scripted：`https://rail.eecs.berkeley.edu/datasets/bridge_release/data/scripted_6_18.zip`
- TFDS：`https://rail.eecs.berkeley.edu/datasets/bridge_release/data/tfds/`

RunPod 上建議先下載較小的 scripted 版：

```bash
bash scripts/runpod/01b_download_bridgedata_scripted.sh
```

若需要 teleop 大包：

```bash
bash scripts/runpod/01c_download_bridgedata_demos.sh
```

建議放在：

```text
/workspace/data/raw/bridgedata_v2
```

> 注意：本專案目前**沒有內建 BridgeData V2 → robomimic HDF5** 轉檔器，因此它屬於「下載教學已補齊、整合待擴充」的來源。

## 5. LIBERO（擴充來源，未內建轉檔）

官方 repo 提供兩種下載方式：
- `python benchmark_scripts/download_libero_datasets.py`
- `python benchmark_scripts/download_libero_datasets.py --use-huggingface`

RunPod 上可用：

```bash
bash scripts/runpod/01d_download_libero.sh
```

預設會下載：
- `libero_100`

若要改成其它套件，可設定：

```bash
export LIBERO_DATASET=libero_goal
bash scripts/runpod/01d_download_libero.sh
```

建議放在：

```text
/workspace/data/raw/libero
```

## 6. CALVIN（擴充來源，未內建轉檔）

官方 repo 提供：

```bash
cd $CALVIN_ROOT/dataset
sh download_data.sh D | ABC | ABCD | debug
```

本專案附上兩個 helper：

```bash
bash scripts/runpod/01e_download_calvin_debug.sh
bash scripts/runpod/01f_download_calvin_full.sh
```

debug 約 1.3GB，先跑 debug 最穩。

建議放在：

```text
/workspace/data/raw/calvin
```

## 7. robomimic（對照 / smoke test 來源）

官方推薦用 `download_datasets.py` 下載，並可指定任務、dataset type 與下載目錄。

本專案附上 helper：

```bash
bash scripts/runpod/01g_download_robomimic_example.sh
```

預設會下載一組小型示例資料到：

```text
/workspace/data/raw/robomimic
```

## 8. REASSEMBLE（參考來源）

REASSEMBLE 官方頁提供 dataset 入口，內容為每 trial 一個 H5 檔，適合之後擴充 contact-rich execution monitoring。由於官方下載入口可能需要頁面互動，本專案目前僅提供文件說明，不提供自動下載腳本。

建議放在：

```text
/workspace/data/raw/reassemble
```

## 9. AHA / FailGen（參考來源）

AHA 官方 repo 提供 Data Generation 與 Visual Instruction Finetuning 路線，但它不是像 DROID / FINO 那樣的固定資料壓縮包下載模式。本專案目前只把它作為 failure augmentation 的參考來源，不做自動下載與轉檔。

## 10. 正式第二章主線的最短路徑

如果你要最省 RunPod 費用、且完全對齊目前程式碼支援範圍，只跑這條：

```bash
bash scripts/runpod/00_bootstrap.sh
bash scripts/runpod/01_download_droid100.sh
bash scripts/runpod/02_convert_droid100.sh
bash scripts/runpod/03_train_droid_debug.sh
bash scripts/runpod/09_demo_droid_debug.sh

# FINO raw 放好之後
bash scripts/runpod/06_generate_fino_manifest.sh
bash scripts/runpod/07_convert_fino.sh
bash scripts/runpod/08_finetune_fino.sh
bash scripts/runpod/10_demo_fino.sh
```


> **重要相容性提醒（RunPod / Python 3.12）：** 若你在 `convert-droid` 遇到 `FieldDescriptor` 沒有 `label` 的錯誤，這通常是 `protobuf>=7` 與 TFDS prepared builder 之間的相容性問題。此專案已加入 runtime shim，且 `tfds` 依賴現在固定為 `protobuf<7` 以降低風險。
